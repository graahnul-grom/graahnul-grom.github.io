6.10 DRC netlister backend

The DRC netlister backend is a simple design rule checking backend by Matt Ettus.

The backend reads the file attribs in the current working directory and makes some simple checks in the schematics specified in the command line:

  • If each of components in the schematics has the attributes specified in the file.
  • If there are nets that have no connected pins.
  • If there are nets that have only one connected pin.

If the attribs file is missing, lepton-netlist warns the user and exits with the status code 1.

Example of the file attribs contents: 

refdes
footprint
value

Example of netlist creation: 

lepton-netlist -g drc -o schematic.drc schematic.sch

Output example: 

S1/U1 Does not have attribute: value
S3/A2 Does not have attribute: footprint
Net S1/unnamed_net4 has only 1 connected pin
