1 Overview

Lepton EDA suite is a set of tools that provide schematic capture, netlisting and export of schematics into several formats, which, in turn, allows simulation via third party tools and producing PCBs.

Programs in the suite

Lepton EDA suite includes tools as follows:

  • lepton-schematic—the main GUI tool for schematic capture.
  • lepton-attrib
  • lepton-netlist—a tool for making netlists from command line
  • lepton-cli—a CLI tool providing support of configuration, export, and shell.
  • lepton-archive—project archivation utility
  • lepton-tragesym—symbol generator
  • lepton-sch2pcb
  • lepton-cli
  • lepton-embed
  • lepton-netlist
  • lepton-pcb_backannotate
  • lepton-refdes_renum
  • lepton-renum
  • lepton-sch2pcb
  • lepton-schdiff
  • lepton-schematic
  • lepton-symcheck
  • lepton-symfix
  • lepton-tragesym
  • lepton-upcfg

The following contributed tools are not covered in this manual. Mostly, those are for working with proprietary, often obsolete, programs, so they are of limited use for most users. Please see corresponding man pages for more information.

  • convert_sym—convert a Viewlogic symbol/schematic to gEDA file format.
  • gmk_sym—create rectangular symbols in gEDA file format from a text file.
  • olib—OrCAD(tm) library part to gEDA .sym converter.
  • sarlacc_schem—convert OrCAD SDT IV files to gEDA file format.
  • smash_megafile—break a Viewlogic megafile into a million little pieces.
  • gnet_hier_verilog.sh—generate a non-flattened hierarchical Verilog netlist.
  • pads_backannotate—process PADS PowerPCB .eco files to backannotate changes to schematics in gEDA file format.
  • sw2asc—converts a SWITCAP2 output file into ASCII data files that other tools can read.
  • sch2eaglepos.sh—read a schematic in gEDA file format and attempt to extract the relative positions of the components in order to generate corresponding MOVE instructions for Eagle.
  • sarlacc_sym—convert OrCAD text libraries to components in gEDA file format.

The following contributed scripts are not covered in this manual either. They are pretty obsolete, non-documented, and usually do trivial things:

  • gschupdate—update some deprecated attributes in old schematics (with version 20020527 or earlier).
  • gsymupdate—update some deprecated attributes in old symbols (with version 20020527 or earlier).
• Compatibility with geda-gaf
